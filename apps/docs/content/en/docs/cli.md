# CLI

The `create-modern-admin` package ships two executable commands:

| Command | Entrypoint | Purpose |
|---|---|---|
| `create-modern-admin` | `bunx create-modern-admin <name>` | Scaffold a brand-new project |
| `modern-admin generate` | `bunx modern-admin generate` | Add system tables to an existing project |

Both commands are implemented in the same binary (`packages/create-modern-admin/src/cli.ts`)
and are also available as a programmatic API.

---

## create-modern-admin — scaffold a new project

Creates a ready-to-run NestJS + Modern Admin project in a new directory.

### Usage

```sh
bunx create-modern-admin <name> [--target ./path]
```

### Arguments

| Argument | Default | Description |
|---|---|---|
| `<name>` | — | Project name (used in `package.json` and as the default directory name) |
| `--target <dir>` | `./<name>` | Directory to write the project into. Created if it does not exist. |
| `-h`, `--help` | — | Print usage and exit |

### What it generates

```
<name>/
  package.json          — dependencies: @modern-admin/core, @modern-admin/nest, NestJS 11
  tsconfig.json
  .gitignore
  .env.example          — PORT, HOST, WEB_ORIGIN, REDIS_URL, DATABASE_URL
  README.md
  src/
    main.ts             — NestJS bootstrap with app.listen()
    app.module.ts       — AppModule with ModernAdminModule.forRoot({ databases: [], resources: [] })
```

### Next steps after scaffolding

```sh
cd <name>
bun install

# Pick an ORM adapter and add it:
bun add @modern-admin/adapter-drizzle drizzle-orm postgres
# — or —
bun add @modern-admin/adapter-prisma @prisma/client

# Pick an auth provider (optional):
bun add @modern-admin/auth-better-auth better-auth

# Run the dev server:
bun run dev
```

### Safety guard

The scaffold command **refuses to write into a non-empty directory**. If the target
directory already exists and contains any files, the command exits with an error.
This prevents accidental overwrites of existing projects.

### Template tokens

All text files in the template are processed through a simple `{{token}}` substitution.
The `{{name}}` token is replaced with the project name you pass on the command line.
This substitution also applies to file and directory **names** — for example a template
file named `{{name}}.config.ts` would be written as `myproject.config.ts`.

## modern-admin generate — add system tables

Adds the Modern Admin framework tables to an existing ORM schema. The command is
**idempotent** — running it multiple times never breaks a schema that already has the
tables in place.

### Usage

```sh
bunx modern-admin generate [options]
```

### Options

| Option | Default | Description |
|---|---|---|
| `--orm prisma\|drizzle` | auto-detected | Force ORM selection. Omit to auto-detect. |
| `--schema <path>` | see below | Override the target schema file. |
| `--cwd <path>` | `process.cwd()` | Working directory for auto-detection and relative paths. |
| `--dry-run` | — | Print the plan without writing any files. |
| `-h`, `--help` | — | Print usage and exit. |

### ORM auto-detection

When `--orm` is not supplied, the command looks for these files relative to `--cwd`:

| File found | Detected ORM |
|---|---|
| `prisma/schema.prisma` | `prisma` |
| `drizzle.config.ts` / `drizzle.config.js` / `drizzle.config.mjs` | `drizzle` |

If neither is found the command exits with an error and asks you to pass `--orm`
explicitly.

## Prisma mode

Reads the canonical model definitions from `@modern-admin/system-prisma` and appends
only the models that are **not yet declared** in your schema. Existing models are never
rewritten — any column customisations you have made survive a re-run.

### Default target file

```
prisma/schema.prisma
```

Override with `--schema path/to/other.prisma`.

### What gets appended

The command parses `model X { … }` blocks from the canonical fragment and compares
their names against the models already present in your schema. Only the missing models
are appended under a clearly labelled comment header:

```prisma
// ─── Modern Admin system tables ──────────────────────────────────────────
// Generated by `bunx modern-admin generate`. Re-run after upgrading
// `@modern-admin/system-prisma` to pick up new tables. Existing models
// are NEVER rewritten — your column tweaks survive.

model MaActionLog {
  id         String   @id
  resourceId String
  action     String
  actorId    String?
  recordId   String?
  payload    Json?
  createdAt  DateTime @default(now())
}

// … more models …
```

### Example output

```
$ bunx modern-admin generate --dry-run

Target (prisma): /myproject/prisma/schema.prisma
Would add (4): MaActionLog, MaWebhook, MaConfig, MaHistory
```

```
$ bunx modern-admin generate

Target (prisma): /myproject/prisma/schema.prisma
Added (4): MaActionLog, MaWebhook, MaConfig, MaHistory
```

On a second run (already up to date):

```
$ bunx modern-admin generate

Target (prisma): /myproject/prisma/schema.prisma
Already up to date — nothing to add.
Skipped (4): MaActionLog, MaWebhook, MaConfig, MaHistory
```

### After running

Run Prisma migrations to apply the new models:

```sh
bunx prisma migrate dev --name add-modern-admin-tables
```

## Drizzle mode

TypeScript is too dynamic to patch safely, so the Drizzle generator takes a different
approach: it writes a **single re-export file** that forwards the table definitions from
`@modern-admin/system-drizzle/pg`. Re-running the command overwrites this file
(it is tagged as auto-generated), so upgrading the framework is as simple as re-running
`generate`.

### Default target file

```
src/db/modern-admin-schema.ts
```

Override with `--schema path/to/target.ts`.

### Generated file content

```ts
// Modern Admin — system tables for Drizzle.
// AUTO-GENERATED by `bunx modern-admin generate` — safe to re-run.
//
// Add to your drizzle schema bag alongside your app tables:
//
//   import * as appSchema from './app-schema'
//   import * as systemSchema from './modern-admin-schema'
//   export const schema = { ...appSchema, ...systemSchema }
//   export const db = drizzle(client, { schema })

export * from '@modern-admin/system-drizzle/pg'
```

### Merging into your schema

Add the generated file to your Drizzle configuration:

```ts
// src/db/schema.ts
import * as appSchema from './app-schema'
import * as systemSchema from './modern-admin-schema'

export const schema = { ...appSchema, ...systemSchema }
```

```ts
// drizzle.config.ts
import { defineConfig } from 'drizzle-kit'

export default defineConfig({
  schema: './src/db/schema.ts',
  out: './drizzle',
  dialect: 'postgresql',
  dbCredentials: { url: process.env.DATABASE_URL! },
})
```

Then generate and apply the migration:

```sh
bunx drizzle-kit generate
bunx drizzle-kit migrate
```

## Programmatic API

Both commands are available as Node/Bun modules for use in scripts and test helpers:

```ts
import { scaffold, generate } from 'create-modern-admin'

// Scaffold a project
const files = await scaffold({
  name: 'my-admin',
  templateDir: '/path/to/template',
  targetDir: '/path/to/output',
  variables: { customToken: 'value' },   // optional extra {{token}} values
})
console.log(`Wrote ${files.length} files`)

// Generate system tables
const result = await generate({
  orm: 'prisma',                          // or 'drizzle' or omit for auto-detect
  schemaPath: 'prisma/schema.prisma',     // optional override
  cwd: process.cwd(),
  dryRun: true,                           // set false to write
})
console.log(result.orm, result.added, result.skipped)
```

### GenerateResult shape

```ts
interface GenerateResult {
  orm: 'prisma' | 'drizzle'
  schemaPath: string      // resolved absolute path of the target file
  added: string[]         // model names (Prisma) or filenames (Drizzle) added
  skipped: string[]       // already present — left untouched
  output: string          // resulting file content after the operation
}
```

## Upgrading system tables

When you upgrade `@modern-admin/system-prisma` or `@modern-admin/system-drizzle`
to a newer version, re-run the generate command to pick up any new tables:

```sh
# After bumping @modern-admin/* versions:
bun install

bunx modern-admin generate
# → adds only the new models; existing ones are untouched

# Prisma: run migration
bunx prisma migrate dev --name upgrade-modern-admin

# Drizzle: generate + apply
bunx drizzle-kit generate && bunx drizzle-kit migrate
```
